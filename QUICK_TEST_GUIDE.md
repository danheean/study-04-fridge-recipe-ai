# 🚀 빠른 테스트 가이드

## 1️⃣ 서버 실행

### 터미널 1: 백엔드
```bash
cd backend
source ../.venv/bin/activate
uvicorn app.main:app --reload --port 8000
```

### 터미널 2: 프론트엔드
```bash
cd frontend
npm run dev
```

## 2️⃣ 브라우저에서 테스트

**URL**: http://localhost:5173

**개발자 도구(F12) 콘솔 열기** - 목 데이터 로그 확인

---

## 3️⃣ 5분 테스트 시나리오

### ✅ Step 1: 이미지 업로드 (2초)
1. 아무 이미지나 업로드 (냉장고 사진 아니어도 됨)
2. "AI가 재료를 분석하고 있습니다..." 확인
3. 2초 후 → 7개 재료 표시 (계란, 우유, 토마토...)

**콘솔 확인**: `📸 [MOCK] 이미지 분석 중...`

---

### ✅ Step 2: 레시피 생성 (2.5초)
1. "레시피 생성하기" 버튼 클릭
2. 로딩 확인
3. 2.5초 후 → 3개 레시피 카드 표시
   - 토마토 치즈 오믈렛
   - 베이컨 야채 볶음밥
   - 크림 야채 스프

**콘솔 확인**: `🍳 [MOCK] 레시피 생성 중...`

---

### ✅ Step 3: 레시피 저장
1. 첫 번째 레시피 "이 레시피 저장하기" 클릭
2. 알림: "레시피가 저장되었습니다!"
3. 버튼 → 초록색 "저장 완료" 체크 아이콘

---

### ✅ Step 4: 프로필 페이지
1. 헤더의 "프로필" 클릭
2. 통계 확인:
   - 저장된 레시피: **1개**
   - 이미지 업로드: **0개**
3. 저장된 레시피 카드 확인

---

### ✅ Step 5: 선호도 설정
1. "선호도 설정" 탭 클릭
2. 값 입력:
   - 식단 제한: `채식`
   - 알레르기: `땅콩, 새우`
3. "선호도 저장" 클릭
4. 알림: "선호도가 저장되었습니다."

---

### ✅ Step 6: 레시피 삭제
1. "저장된 레시피" 탭으로 돌아가기
2. 휴지통 아이콘 클릭
3. 확인 다이얼로그 → "확인"
4. 레시피 삭제됨
5. 통계: **1개 → 0개**

## 💡 문제 해결

### 백엔드 연결 오류
```bash
# 기존 프로세스 종료
pkill -f uvicorn

# 다시 시작
cd backend
source ../.venv/bin/activate
uvicorn app.main:app --reload --port 8000
```

### 프론트엔드 오류
```bash
cd frontend
npm install
npm run dev
```

### 목 데이터가 작동하지 않음
`frontend/src/services/api.js` 파일 확인:
```javascript
const USE_MOCK_DATA = true; // 이 값이 true인지 확인
```
